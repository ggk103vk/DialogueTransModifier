DialogueTransModifier - Beta v0.1 releases



이 프로그램은 기본적으로 whoami님의 "쯔꾸르 대사 추출 밒 삽입 프로그램 MVMZ" 게시글에 소개 되어 있는

「텍스트 추출 및 체인지 MVMZ (line number)」 

라는 프로그램을 사용하여 추출 된 텍스트 파일의 번역 보조를 위해 제작되었습니다.

따라서 다른 추출 프로그램으로 추출 된 경우 
시작화면의 '범용 텍스트 파일' 항목에 체크 하고 실행하시길 바랍니다.


=====[   시작 화면 설명   ]======

작업 파일 : [        경로를 표시하는 공간        ] [ 불러오기 ]
- 작업을 위한 파일의 경로를 찾아 불러옵니다.

참조 파일[기계번역] : [        경로를 표시하는 공간        ] [ 불러오기 ] 
- 비교를 위한 파일의 경로를 찾아 불러옵니다. 
- 기계번역을 통으로 돌린 텍스트 파일을 불러와 비교하실때 이용하시면 됩니다.
- 해당 파일을 불러오게 되면 기본적으로 입력 텍스트 박스란에 기본적으로 비교용 파일의 대사 줄을 가져옵니다.

필터링 문자열 추가 : [      텍스트를 입력하는 공간      ]
- 텍스트 파일에서 추출하지 않고 싶은 행의 시작 글자를 입력합니다.
- 기본적으로 주석 행을 제외하기 위해 '//' 와 '#'가 적용되어 있습니다. 
- 텍스트 파일의 구조를 파악하기 전이라면 사용을 추천하지 않으며, 추가로 시스템 변수가 확실한 것 같다 싶으신 문자열을 넣으시면 될 것 같습니다.
 
캐릭터 이름 필터링 : [      텍스트를 입력하는 공간      ] [ 불러오기 ]
- 일반적으로는 Merge 문자열에서 캐릭터 이름을 제외하기 위해 사용됩니다.
- 범용 텍스트 파일의 경우 캐릭터의 이름을 기준으로 대사를 나누는 기능을 위해 활용됩니다.
- 불러오기 버튼을 통해 .txt 나 .csv로 된 파일을 받아올 수 있습니다.
- 이름을 나누는 기준은 ', '과 행으로 

.csv파일은 기본적으로 한 칸에 이름 한 개씩 넣어주시면 됩니다.
ex)
이름1, 이름2
이름3
다만 공백 문자가 삽입 될 수 있으므로 무조건 이름과 이름 사이에 빈 칸이 없도록 작성해 주세요

.txt파일은 , 를 구분자로 넣으시거나 줄바꿈을 통해 구분되어 입력됩니다.
ex)
이름1, 이름2, 이름3
이름1
이름2
이름3
형식으로 작성하시면 됩니다. 두가지 경우가 섞여도 작동합니다. 다만 위와 마찬가지로 ', '와 ', '사이나 행과 행 사이를 빈칸으로 두어선 안됩니다.

□범용 텍스트 파일 [               작업 시작               ]  버전 코드

□범용 텍스트 파일
- 범용 텍스트 파일에 체크하시면 라인넘버가 없는 텍스트 파일을 불러오실 수 있습니다.
- 기본적으로는 해제 되어 있으며 해제 상태에서 "라인넘버가 없는 텍스트 파일"을 불러올 경우 다음 출력 화면에 아무것도 표시되지 않을겁니다
- 체크 되어 있는 상태에서 작업 시작을 누르면 해당 텍스트 파일에 임의적으로 라인 넘버를 부여하여 보여줍니다.
- 저장 시에는 부여하였던 라인 넘버를 지우고 저장하기에 걱정하지 않으셔도 됩니다.
- 체크 상태에서 실행하먼 기본적으로 4줄씩 끊어서 보여주고 캐릭터 이름 행을 확인할경우 해당 행을 기준으로 한번 끊어서 불러옵니다.

[ 작업 시작 ]
작업 시작 버튼을 누르면 작업 공간 화면으로 넘어갑니다.

=====[   작업 화면 설명   ]======
□저장시 백업 [ 저장 ][ 종료 ]                              [ 백업 ]

□저장 시 백업
- 체크 되어 있는 경우[기본상태]
	파일을 저장하기 전 수정을 반영하지 않은 백업 파일을 저장합니다. 이후 파일을 저장합니다.
- 체크 해제 된 경우
	백업을 저장하지 않고 바로 파일을 저장합니다

[ 저장 ]
- 저장을 실행합니다. 위를 참고하세요 
- 단축키 : Ctrl+S

[ 종료 ]
- 수정 시 무조건 저장 의사를 확인 한 후 종료합니다
- 종료시 프로그램 실행 위치에 마지막 작업 라인 위치를 저장하는 SavePoint.last_line 파일을 생성합니다.(실행하기 전에는 파일이 생성 되지 않으며 해당 파일 삭제시 '마지막 작업 위치 기록' 삭제 가능)
- 윈도우 창의 닫기[ X ]를 누르실 때에도 이 버튼과 같은 동작을 실행합니다
- 단축키 : Alt+F4

[ 백업 ]
- 수정 내역을 반영하지 않은 원본 텍스트 파일을 저장합니다. 
- 기본적으로 저장시 백업이 체크 되어 있으므로 거의 사용하지 않을 버튼이지만, 
- 만일을 대비해 작업을 시작하자마자 백업 하시겠다는 분들을 위해 남겨두었습니다.

┌─────────────────────────────────┐
│                        비교용 대사 출력                        │
└─────────────────────────────────┘
- 비교용 텍스트 파일을 불러온 경우 화면 상에 출력됩니다.

┌─────────────────────────────────┐
│                        작업용 대사 출력                        │
└─────────────────────────────────┘
- 작업용 텍스트 파일의 대사 내용을 화면 상에 출력됩니다.

두 박스 모두 기본적으로
ex)
00001:[Alice] //텍스트 추출 및 체인지 MVMZ로 추출한 텍스트 파일의 경우 대부분 첫 번째로 표시되는 열이 이름입니다.
00002:Hello World!
00003:Have a nice day!
Merge:[Alice]Hello World!Have a nice day!
와 같이 출력 합니다. 다만 

위에서 캐릭터 이름 필터링 항목에
'[Alice]'라는 문자열을 입력해 두었다면
ex)
00001:[Alice]
00002:Hello World!
00003:Have a nice day!
Merge:Hello World!Have a nice day!
와 같이 출력 하게 됩니다.

Merge: 라인은 중간에 끊겨서 위 아래로 나누어진 대사를 보기 쉽게하기 위해 출력하도록 작성되었으며
이후 등장할 [ 합쳐서 보기 ] 버튼을 통해 ON/OFF를 조정할 수 있습니다.

라인 넘버 : 현재 라인 넘버 / 마지막 줄 라인넘버 [ 이동할 라인 번호 ] [ 이동 ] [ 합쳐서 보기 ] ON

[ 이동 ] 
- 이동 버튼을 누르면 앞의 텍스트 박스에 적은 이동할 라인 번호 위치로 이동합니다
이때 꼭 00001 과 같이 모양을 맞추지 않으셔도 됩니다.
1을 적어두고 이동 버튼을 누르셔도 00001번 라인으로 이동합니다.

[ 합쳐서 보기 ] ON / OFF
- Merge: 라인을 출력 하는 것을 ON / OFF 하는 버튼입니다.
- 기본적으로는 ON 상태 입니다.

                     □Enter로 바로 수정 적용
┌─────────────────────────────────┐
│                        수정할 대사 작성                        │
└─────────────────────────────────┘
[ 이전 페이지 ] [ 이전 ] [   수정   ] [ 다음 ] [ 다음 페이지 ]

□Enter로 바로 수정 적용
- 기본적으로는 체크 해제 되어 있습니다
- 이 항목에 체크 하신 경우 Enter 키를 누르는 것만으로도 [ 수정할 대사 작성 ]에 적혀있는 대사를 바로 수정 적용하실 수 있습니다
- "주의" : 수정 적용을 한다고 해서 수정 내역이 저장되는 것은 아닙니다. 저장 버튼(Ctrl+S)를 눌러야만 수정 내역이 저장됩니다.

[ 수정할 대사 작성 ] 
- 해당 박스에는 기본적으로 "현재 작업하고 있는 파일"의 같은 라인 위치의 대사를 받아와 보여주고 있습니다.
- 비교용 파일을 불러온 경우 "비교용 파일"의 현재 작업하고 있는 라인 위치의 대사를 받아옵니다.

[ 이전 페이지 ] [ 다음 페이지 ]
- 대사를 한 묶음 단위로 이동합니다.
- "주의" : [수정할 대사 작성 박스]의 내용이 대사 파일에 반영되지 않고 이동 하므로 
  	     수정을 적용하기 위해 텍스트를 입력하는 중이라면 누르지 않는 것을 추천합니다.
- 단축키 : Shift+(Home | End) //(키패드 포함)

[ 이전 ] [ 다음 ]
- 대사를 한 줄 단위로 이동합니다.
- "주의" : [수정할 대사 작성 박스]의 내용이 대사 파일에 반영되지 않고 이동 하므로 
  	     수정을 적용하기 위해 텍스트를 입력하는 중이라면 누르지 않는 것을 추천합니다.
- 단축키 : Shift+(PgUp | PgDn) //(키패드 포함)

[ 수정 ]
- 입력한 텍스트로 현재 작업 중인 라인의 대사를 교체합니다.
- 단축키 : Shift+Enter, Enter로 바로 수정 적용에 체크되어 있을때 Enter

기타 단축키들
F5, Ctrl+R = 새로 고침 -> 현재 라인 넘버 위치의 대사를 출력창과 입력창에 다시불러옴
Ctrl+F = "주의" : [수정할 텍스트 입력] 상자의 모든 텍스트를 삭제함


-----{   주의사항   }-----
위에서 말한 것 처럼 이 프로그램은 「텍스트 추출 및 체인지(line number)」로 추출한 텍스트 파일의 수정작업을 보조하기 위해 제작 되었습니다.

'https://github.com/gramedcart/tsukuru_extractor/releases' 페이지의
「tsukuru extractor」 프로그램으로 추출하신 분은 파일 구조가 제가 상정한 것과는 차이가 있으므로 수정 시 주의를 요합니다.
